# B2B-приложение ООО ТПК «Полимер-Пласт»

Мобильное приложение для поддержки B2B-продаж ПВХ-пластиката с интеграцией
интеллектуального чат-бота. Выпускная квалификационная работа.

**Стек:** React Native (Expo) · Node.js + Express · PostgreSQL · JWT + bcrypt · PDFKit · локальный чат-бот.

Приложение **адаптивное**: на телефоне (Expo Go) — мобильная вёрстка с нижними вкладками;
в браузере на широком экране (менеджеры) — верхняя панель навигации и многоколоночные
раскладки (каталог: фильтры · карточки · «Ваш заказ»; карточка товара: фото · описание · заказ).

---

## Запуск одним кликом (Windows)

В корне есть `.bat`-файлы (см. `ЧИТАЙ-МЕНЯ-запуск.txt`):

- **`ЗАПУСТИТЬ-ВСЁ.bat`** — поднимает API и приложение в двух окнах;
- **`ОСТАНОВИТЬ-ВСЁ.bat`** — останавливает оба сервера;
- **`НАСТРОЙКА-первый-запуск.bat`** — один раз на новой машине (зависимости, БД, данные).

После запуска: в окне «Приложение» нажмите `w` (веб) или сканируйте QR в Expo Go.

## Быстрый старт

### Требования
- Node.js 18+ (проверено на 24)
- PostgreSQL 14+ (проверено на 17). Пользователь `postgres` / пароль `postgres`, порт `5432`.
- Для телефона — приложение **Expo Go** (Android/iOS).

### 1. Backend

```bash
cd backend
npm install
copy .env.example .env          # Windows (или cp на macOS/Linux)
npx prisma migrate dev          # создать таблицы
npm run db:setup                # импорт каталога + компаний, демо-данные, каталог-PDF
npm run dev                     # запуск API на http://localhost:4000
```

Проверка: открыть http://localhost:4000/api/health → `{"status":"ok"}`.
Сквозной тест всех сценариев: `npm run smoke`.

### 2. Mobile (Expo)

```bash
cd mobile
npm install
npx expo start                  # откроется Metro; нажмите 'w' для веб, или
                                # отсканируйте QR в Expo Go на телефоне
```

> Адрес API определяется автоматически: для телефона подставляется IP компьютера,
> для веб-версии и симулятора — `localhost`. При необходимости поправьте `mobile/src/config.js`.
> Телефон и компьютер должны быть в одной Wi-Fi сети.

## Реализованный функционал (соответствие требованиям ВКР)

| № | Требование | Где реализовано |
|---|------------|-----------------|
| 1 | Просмотр каталога продукции (+ единый файл каталога) | Каталог → карточка; PDF-каталог (`db:catalog`) |
| 2 | Отдельная таблица цен + процедура обновления (раз в 2 нед.) | модель `Price`/`PriceUpdate`; экран «Цены» (менеджер) |
| 3 | Формирование КП-счёта; объём в кг + регион; образцы 5/10/20/30 кг | Корзина → КП (PDF); карточка товара |
| 4 | Счёт ТК (только менеджер) + платёжки (только клиент) | загрузка документов с разграничением по ролям |
| 5 | Автоотмена при неоплате в течение 72 ч | планировщик `jobs/autocancel.js` |
| 6 | Регистрация: ФИО, должность, ООО/ИП, реквизиты; компания отдельно | экран регистрации; модели `User` + `Company` |
| 7 | Лояльность: 200 т → 2%, 500 т → 5% за 2 года (на всю компанию) | `services/loyalty.js`; экран «Профиль» |
| 8 | Отслеживание заказа | таймлайн истории статусов в карточке заказа |
| 9 | Страница сформированных заказов | вкладка «Заказы» |
| 10 | Характеристики (RAL, Шор, темп. хрупкости, ПТР) + метки применения | карточка продукции; 12 меток применения |
| + | Интеллектуальный чат-бот (подбор марок, FAQ, эскалация) | вкладка «Помощник»; `services/chatbot.js` |
| + | Рекламации по заказу | экран «Рекламация» |

Подробнее об архитектуре — [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
